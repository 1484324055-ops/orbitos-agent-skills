# Privacy Guide

These skills were extracted from a real personal knowledge workflow. The public versions are intentionally sanitized.

## Do Not Commit

- customer names
- private chat logs
- local vault paths
- personal voice samples
- API keys or cookies
- unpublished business strategies
- screenshots with private data

## Recommended Pattern

Keep public skills generic. Let users provide private context at runtime.

Example:

```text
Use the write skill. Here is my private tone guide for this draft only: ...
```

Do not hard-code that tone guide into the public skill.
